# Documentation stack

This repository splits source-of-truth artifacts by purpose:

- `docs/proposals/`: why work matters.
- `docs/specs/`: behavior contracts and proof requirements.
- `docs/adr/`: durable architectural decisions.
- `docs/status/SUPPORT_TIERS.md`: claim-to-proof mapping.
- `docs/handoffs/`: closeout records.
- `docs/templates/`: required authoring templates.

Cross-artifact linkage is tracked in `policy/doc-artifacts.toml`.
